🔍 Nmap Features (Command Line Options) - पूरी जानकारी उदाहरण सहित

यहाँ हर Nmap कमांड का syntax, example और यह बताया गया है कि उसका उपयोग करने से क्या होता है।

🎯 TARGET SPECIFICATION (लक्ष्य निर्दिष्ट करना)

CommandSyntax & Exampleक्या होता है? (What it does)
--exclude--exclude <host1,host2>
nmap 192.168.1.1/24 --exclude 192.168.1.5,192.168.1.10		स्कैन से कुछ specific hosts को बाहर कर देता है।
--excludefile--excludefile <file>
nmap 192.168.1.1/24 --excludefile exclude.txt		एक फ़ाइल से hosts को पढ़कर उन्हें स्कैन से बाहर करता है।
-iL-iL <inputfile>
nmap -iL targets.txt		किसी फ़ाइल से target hosts की list पढ़ता है।

🕵️ HOST DISCOVERY (होस्ट खोज)

CommandSyntax & Exampleक्या होता है?
-sL-sL <target>
nmap -sL 192.168.1.1/24		List Scan – बिना कोई packet भेजे सिर्फ targets की लिस्ट बनाता है।
-sn-sn <target>
nmap -sn 192.168.1.1/24		Ping Scan – केवल यह पता करता है कि कौन से hosts alive हैं (port scan नहीं करता)।
-Pn-Pn <target>
nmap -Pn 192.168.1.1		मानता है कि सभी hosts alive हैं – ping को skip करता है (firewall वाले hosts के लिए)।
-PS-PS <portlist>
nmap -PS22,80,443 192.168.1.1		TCP SYN ping – specified ports पर SYN packet भेजता है।
-PA-PA <portlist>
nmap -PA22,80 192.168.1.1		TCP ACK ping – ACK packet भेजता है।
-PU-PU <portlist>
nmap -PU53 192.168.1.1		UDP ping – UDP packet भेजता है।
-PY-PY <portlist>
nmap -PY22 192.168.1.1		SCTP ping – SCTP INIT packet भेजता है।
-PO-PO <protocol list>
nmap -PO1,2,4 192.168.1.1		IP Protocol ping – specified IP प्रोटोकॉल के packets भेजता है।
--traceroute--traceroute <target>
nmap --traceroute 8.8.8.8		Target तक पहुँचने के लिए network path (hops) दिखाता है।
--system-dns--system-dns <target>
nmap --system-dns google.com		DNS resolution के लिए system के DNS resolver का उपयोग करता है।

⚙️ SCAN TECHNIQUES (स्कैन तकनीकें)

CommandSyntax & Exampleक्या होता है?
-sS-sS <target>
nmap -sS 192.168.1.1		TCP SYN Stealth Scan – आधा कनेक्शन बनाता है, तेज़ और कम दिखता है।
-sT-sT <target>
nmap -sT 192.168.1.1		TCP Connect Scan – पूरा TCP handshake करता है।
-sA-sA <target>
nmap -sA 192.168.1.1		TCP ACK Scan – firewall rules का पता लगाने के लिए ACK packets भेजता है।
-sW-sW <target>
nmap -sW 192.168.1.1		TCP Window Scan – window size के आधार पर port status बताता है।
-sM-sM <target>
nmap -sM 192.168.1.1		TCP Maimon Scan – FIN/ACK packets भेजता है।
-sU-sU <target>
nmap -sU 192.168.1.1		UDP Scan – UDP ports के लिए स्कैन करता है (धीमा हो सकता है)।
-sN-sN <target>
nmap -sN 192.168.1.1		TCP Null Scan – बिना किसी flag के packet भेजता है।
-sF-sF <target>
nmap -sF 192.168.1.1		TCP FIN Scan – सिर्फ FIN flag से packet भेजता है।
-sX-sX <target>
nmap -sX 192.168.1.1		TCP Xmas Scan – FIN, PSH, URG flags सभी set होते हैं।
--scanflags--scanflags <flags>
nmap --scanflags SYN,ACK 192.168.1.1		कस्टम TCP flags के साथ स्कैन करता है।
-sZ-sZ <target>
nmap -sZ 192.168.1.1		SCTP Cookie Echo Scan (SCTP protocol के लिए)।
-sI-sI <zombie>
nmap -sI 192.168.1.50:80 192.168.1.1		Idle Scan – zombie host का उपयोग करके बिना पहचाने स्कैन करना।
-sO-sO <target>
nmap -sO 192.168.1.1		IP Protocol Scan – कौन से IP प्रोटोकॉल (TCP, UDP, ICMP etc.) support हैं।
-b-b <ftp relay>
nmap -b ftp://user:pass@server:21		FTP Bounce Scan – FTP server के माध्यम से स्कैन करता है (पुरानी तकनीक)।

🔌 PORT SPECIFICATION AND SCAN ORDER

CommandSyntax & Exampleक्या होता है?
-p-p <port ranges>
nmap -p22,80,443 192.168.1.1
nmap -p U:53,111,T:21-25,80		Specified ports को ही scan करता है। UDP/TCP अलग-अलग specify कर सकते हैं।
--exclude-ports--exclude-ports <port ranges>
nmap -p 1-1000 --exclude-ports 22,80		कुछ ports को scan से बाहर कर देता है।
-F-F <target>
nmap -F 192.168.1.1		Fast scan – सिर्फ 100 सबसे common ports scan करता है।
-r-r <target>
nmap -r 192.168.1.1		Ports को random क्रम में नहीं, बल्कि sequential order में scan करता है।
--top-ports--top-ports <number>
nmap --top-ports 10 192.168.1.1		सबसे popular ‘number’ ports को scan करता है।

🔧 SERVICE/VERSION DETECTION

CommandSyntax & Exampleक्या होता है?
-sV-sV <target>
nmap -sV 192.168.1.1		Open ports पर चल रहे service और उनके version का पता लगाता है।
--version-intensity--version-intensity <0-9>
nmap -sV --version-intensity 5 192.168.1.1		Version detection की गहराई (intensity) set करता है। 9 = सबसे गहरा।
--version-all--version-all <target>
nmap -sV --version-all 192.168.1.1		हर port पर हर probe भेजता है (intensity 9 के बराबर)।
--version-trace--version-trace <target>
nmap -sV --version-trace 192.168.1.1		Version detection के दौरान भेजे गए packets का detailed log दिखाता है।

📜 SCRIPT SCAN (NSE Scripts)

CommandSyntax & Exampleक्या होता है?
-sC-sC <target>
nmap -sC 192.168.1.1		Default scripts चलाता है (equivalent to --script=default)।
--script--script=<lua scripts>
nmap --script=http-title,ssh-auth-methods 192.168.1.1		Specified Lua scripts को चलाता है (category या नाम से)।
--script-args--script-args=<args>
nmap --script=http-title --script-args http.useragent="Mozilla"		Scripts को arguments pass करता है।
--script-args-file--script-args-file <filename>
nmap --script-args-file args.txt		किसी फ़ाइल से script arguments पढ़ता है।
--script-trace--script-trace <target>
nmap --script-trace 192.168.1.1		Script द्वारा भेजे/प्राप्त packets को दिखाता है।
--script-updatedb--script-updatedb
nmap --script-updatedb		Nmap script database को update करता है।

🖥️ OS DETECTION

CommandSyntax & Exampleक्या होता है?
-O-O <target>
nmap -O 192.168.1.1		Target के operating system का पता लगाने की कोशिश करता है।
--osscan-limit--osscan-limit <target>
nmap -O --osscan-limit 192.168.1.1		OS detection सिर्फ उन hosts पर करता है जो कम से कम एक open और एक closed port दिखाते हैं।
--osscan-guess--osscan-guess <target>
nmap -O --osscan-guess 192.168.1.1		अगर exact match न मिले तो भी ज़्यादा से ज़्यादा guess करता है।

⏱️ TIMING AND PERFORMANCE

CommandSyntax & Exampleक्या होता है?
-T<0-5>-T<0-5>
nmap -T4 192.168.1.1		Timing template: 0=paranoid, 1=sneaky, 2=polite, 3=normal, 4=aggressive, 5=insane
--min-hostgroup / --max-hostgroup--min-hostgroup <size> --max-hostgroup <size>
nmap --min-hostgroup 10 --max-hostgroup 50 192.168.1.1/24		Parallel में scan किए जाने वाले hosts के समूह का size तय करता है।
--max-retries--max-retries <tries>
nmap --max-retries 3 192.168.1.1		Port scan के दौरान retries की अधिकतम संख्या।
--min-rtt-timeout
--max-rtt-timeout
--initial-rtt-timeout		--min-rtt-timeout <time>
nmap --initial-rtt-timeout 100ms --max-rtt-timeout 300ms		Round-trip time timeout values को manually set करता है।
--host-timeout--host-timeout <time>
nmap --host-timeout 5m 192.168.1.1		एक host पर scan करने का maximum समय, उसके बाद छोड़ देता है।
--scan-delay / --max-scan-delay--scan-delay <time>
nmap --scan-delay 1s 192.168.1.1		दो probes के बीच delay डालता है (IDS evasion के लिए)।
--min-rate / --max-rate--min-rate <number> --max-rate <number>
nmap --min-rate 100 --max-rate 500 192.168.1.1		प्रति सेकंड packets की दर (rate) को नियंत्रित करता है।

🛡️ FIREWALL/IDS EVASION AND SPOOFING

CommandSyntax & Exampleक्या होता है? -f-f <target>
nmap -f 192.168.1.1Packet fragmentation – packets को छोटे fragments में तोड़ता है (IDS बाईपास)। --mtu--mtu <val>
nmap --mtu 8 192.168.1.1MTU (Maximum Transmission Unit) को specified value पर set करता है। -D-D <decoy1,decoy2,ME>
nmap -D 10.0.0.1,10.0.0.2,ME 192.168.1.1Decoy hosts – असली IP को छुपाने के लिए झूठे IPs भेजता है। -S-S <IP_Address>
nmap -S 10.0.0.5 192.168.1.1Source IP address spoof करता है। -e-e <iface>
nmap -e eth0 192.168.1.1Specified network interface का उपयोग करता है। -g / --source-port-g <portnum>
nmap -g 53 192.168.1.1Source port number को fake करता है (e.g., port 53 DNS जैसा दिखाने के लिए)। --proxies--proxies <url1,url2>
nmap --proxies http://proxy1:8080,http://proxy2:8080 192.168.1.1HTTP/SOCKS4 proxy chain के माध्यम से स्कैन करता है। --data--data <hex string>
nmap --data 0xDEADBEEF 192.168.1.1Raw binary data (hex) को packets में जोड़ता है। --data-string--data-string <string>
nmap --data-string "HELLO" 192.168.1.1Text string को packets में शामिल करता है। --data-length--data-length <num>
nmap --data-length 200 192.168.1.1Random data of given length को packets में जोड़ता है। --ip-options--ip-options <options>
nmap --ip-options "L 10.0.0.1" 192.168.1.1IP header में options जोड़ता है (जैसे Loose Source Routing)। --ttl--ttl <val>
nmap --ttl 50 192.168.1.1IP TTL (Time To Live) value को set करता है। --spoof-mac--spoof-mac <mac/prefix/vendor>
nmap --spoof-mac 00:11:22:33:44:55MAC address को spoof करता है (vendor name भी दे सकते हैं जैसे Apple, Cisco)। --badsum--badsum <target>
nmap --badsum 192.168.1.1गलत TCP/UDP checksum वाले packets भेजता है (कुछ firewalls को पहचानने के लिए)।

📄 OUTPUT (आउटपुट)

CommandSyntax & Exampleक्या होता है?
-oN-oN <file>
nmap -oN result.txt 192.168.1.1		Normal output को फ़ाइल में save करता है।
-oX-oX <file>
nmap -oX result.xml 192.168.1.1		XML format में output save करता है।
-oS-oS <file>
nmap -oS result.script 192.168.1.1		ScRipT KIddIe style output (1337 speak में)।
-oG-oG <file>
nmap -oG result.gnmap 192.168.1.1		Grepable output – tools के साथ parse करने के लिए।
-oA-oA <basename>
nmap -oA scan 192.168.1.1		तीनों formats (.nmap, .xml, .gnmap) एक साथ generate करता है।
-v-v <target>
nmap -v 192.168.1.1		Verbose output – ज़्यादा जानकारी दिखाता है।
-d-d <target>
nmap -d 192.168.1.1		Debugging output (बहुत detailed)।
--reason--reason <target>
nmap --reason 192.168.1.1		बताता है कि port open/closed क्यों है (जैसे SYN-ACK मिला, RST मिला आदि)।
--open--open <target>
nmap --open 192.168.1.1		सिर्फ open (या open|filtered) ports दिखाता है, closed नहीं।
--packet-trace--packet-trace <target>
nmap --packet-trace 192.168.1.1		हर भेजा और प्राप्त packet को दिखाता है।
--iflist--iflist
nmap --iflist		Available network interfaces और routes की list दिखाता है।
--append-output--append-output <file>
nmap -oN result.txt --append-output 192.168.1.1		मौजूदा फ़ाइल में output जोड़ता है (overwrite नहीं करता)।
--stylesheet--stylesheet <path/URL>
nmap -oX result.xml --stylesheet https://nmap.org/data/nmap.xsl		XML output के लिए custom XSLT stylesheet सेट करता है।
